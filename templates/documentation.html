Overview

The Expense Tracker web application allows users to track and manage their daily expenses. Users can register, log in, add expenses to specific dates, view expenses, and apply filters to view expenses by categories or date ranges. The application provides a user-friendly interface with a calendar view and modal forms for adding and editing expenses.

1. Registration Page

Purpose: Allows users to register for an account.

  • Form Fields: Username, Password, Email.
  • Action: Upon successful registration, users are redirected to the login page.

2. Login Page

Purpose: Authenticates users to allow access to the Expense Tracker.

  • Form Fields: Username, Password.
  • Action: On successful login, users are redirected to the home page where they can start adding and viewing expenses.

3. Home Page

Purpose: Displays the main interface of the expense tracker, providing navigation links for adding expenses and viewing expenses.

  • Navigation: Links to "Add Expense" and "View Expenses" pages.
  • Call to Action: A button leading users to start adding expenses.

4. Add Expense Page

Purpose: Allows users to add expenses on specific dates using a calendar interface.

  • Calendar: Displays a calendar with clickable dates. The current month is shown by default.
  • Expense Form: When a user clicks on a date, an expense form modal is displayed, where they can input the total amount, category, and description of the expense.
  • Submit Button: After filling the form, users can submit the expense, which will be saved to the database.
  • View Button: A button in the modal redirects users to the "View Expenses" page for that specific date.

5. View Expenses Page

Purpose: Displays a summary of all expenses and allows users to filter expenses by date range and category.

  • Filters: Users can filter expenses by category and date range.
  • Expense Summary: Displays the total expenses for the selected filters.
  • Expense Table: Displays a table with all expenses, including the date, description, category, and amount.
  • No Record Found: If there are no expenses for the selected filters, a "No record found" message is displayed.

6. Expense Details Page

Purpose: Displays a detailed view of the expenses for a particular date, allowing users to see specific expense details for that day.

  • Details: Displays the breakdown of each expense added for the selected date.
